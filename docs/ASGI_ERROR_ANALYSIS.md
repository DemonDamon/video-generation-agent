# ASGI 错误分析与解决方案

## 🔍 错误信息

```
2026-02-28 17:17:00 ERROR:    ASGI callable returned without completing response.
```

---

## 📊 错误含义

### **ASGI 是什么？**
ASGI (Asynchronous Server Gateway Interface) 是 Python 异步 Web 应用的标准接口。

### **错误解释**
```
ASGI callable returned without completing response
```

**含义：** ASGI 应用（您的 Agent）在完成响应之前就返回了，导致：
- 流式响应未正确关闭
- 客户端收到不完整的响应
- 连接异常中断

---

## 🎯 根本原因分析

### 根据您的场景，可能的原因：

### 1️⃣ **视频生成时间过长，超过超时限制**

**证据：**
```python
# config/agent_llm_config.json
timeout: 600  # 600 秒（10 分钟）

# 实际视频生成时间
场景1: 280 秒（4.7 分钟）
场景2: 110 秒（1.8 分钟）
总计: 391 秒（6.5 分钟）
```

**问题：**
- 虽然总时间未超过 600 秒，但接近限制
- ASGI 服务器（uvicorn）可能有默认的连接超时
- 长时间无响应可能导致连接断开

### 2️⃣ **流式响应中断**

**流程：**
```
用户请求
  ↓
Agent 开始流式响应（streaming=True）
  ↓
调用视频生成工具（长时间阻塞）
  ↓
ASGI 连接超时或客户端断开
  ↓
Agent 继续执行，但连接已断开
  ↓
尝试发送响应时失败
  ↓
ASGI callable returned without completing response
```

### 3️⃣ **异常未正确处理**

如果工具调用过程中出现异常，可能导致：
- 响应提前结束
- 流式响应未正确关闭

---

## 💡 解决方案

### 方案 1：增加超时配置（推荐）

修改 `config/agent_llm_config.json`：

```json
{
    "config": {
        "model": "doubao-seed-1-8-251228",
        "temperature": 0.7,
        "top_p": 0.9,
        "max_completion_tokens": 10000,
        "timeout": 1800,  // 增加到 30 分钟
        "thinking": "disabled"
    }
}
```

**同时需要配置 ASGI 服务器超时：**

如果使用 uvicorn，需要设置 `--timeout-keep-alive` 参数：

```bash
uvicorn src.main:app --timeout-keep-alive 1800
```

### 方案 2：使用异步视频生成（推荐）

**原理：**
- 视频生成改为异步模式
- Agent 立即返回任务 ID
- 用户通过任务 ID 查询进度

**优点：**
- ✅ 不会阻塞 ASGI 连接
- ✅ 用户体验更好（立即响应）
- ✅ 避免超时问题

**实现步骤：**

1. 修改工具为异步模式
2. Agent 返回任务 ID
3. 提供查询进度的工具

### 方案 3：优化视频生成策略

**减少单次生成的时长：**

```
当前：每次生成 5 秒视频
优化：改为每次生成 3 秒视频
```

**减少场景数量：**

```
当前：3-5 个场景
优化：2-3 个场景
```

### 方案 4：添加异常处理和重试机制

在工具调用中添加更完善的异常处理：

```python
try:
    video_url, response, last_frame = client.video_generation(...)
except Exception as e:
    logger.error(f"视频生成异常: {e}")
    # 返回明确的错误信息
    return json.dumps({
        "error": str(e),
        "status": "failed",
        "message": "视频生成超时或失败，请稍后重试"
    })
```

## 🔧 推荐的综合解决方案

### **短期方案（立即可用）：**

1. **增加超时配置**
   ```json
   "timeout": 1800  // 30 分钟
   ```

2. **减少单次生成时长**
   ```
   单场景时长: 3-4 秒（而不是 5 秒）
   场景数量: 最多 3 个
   ```

3. **优化 System Prompt**
   ```
   建议用户生成 2-3 个场景
   提前告知可能需要等待 5-10 分钟
   ```

### **长期方案（最佳实践）：**

**实现异步视频生成：**

1. 创建异步任务管理器
2. Agent 立即返回任务 ID
3. 提供进度查询接口
4. 支持任务取消和重试

**架构：**
```
用户请求
  ↓
Agent 创建任务，返回 task_id
  ↓
后台异步生成视频
  ↓
用户通过 task_id 查询进度
  ↓
生成完成后获取视频 URL
```
